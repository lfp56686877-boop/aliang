# 用AI写代码的正确姿势：prompt engineering for developers

> 用AI写代码不是"复制粘贴"，而是"有效沟通"。本文教你如何用正确的提示词让AI生成高质量代码。

## 一、AI写代码的常见问题

直接让AI写代码，通常会出现：
- 代码能跑但不是最优解
- 缺少错误处理
- 不符合项目规范
- 安全漏洞

核心原因：你给的提示词不够具体。

## 二、AI写代码的提示词框架

### 基础框架
```
语言/框架 + 功能需求 + 输入输出 + 约束条件 + 代码风格
```

### 示例
```
用Python实现一个函数：
- 功能：计算两个日期之间的工作日数量（排除周末）
- 输入：两个datetime对象
- 输出：整数（工作日天数）
- 约束：不使用第三方库
- 风格：遵循PEP8，添加类型注解和文档字符串
```

## 三、不同场景的提示词

### 1. 实现功能
```
用TypeScript实现一个防抖函数：
- 支持泛型
- 支持取消操作
- 支持立即执行选项
- 添加完整的JSDoc注释
```

### 2. 修复Bug
```
以下代码在处理空数组时报错：
[粘贴代码]
请：
1. 找出Bug原因
2. 修复代码
3. 添加边界条件处理
4. 说明修复原因
```

### 3. 代码审查
```
请审查以下代码：
[粘贴代码]
审查维度：
1. 安全性（SQL注入、XSS等）
2. 性能（算法复杂度、内存使用）
3. 可读性（命名、结构、注释）
4. 最佳实践（设计模式、SOLID原则）
```

### 4. 重构优化
```
请重构以下代码：
[粘贴代码]
目标：
1. 提高可读性
2. 减少重复代码
3. 符合[框架]最佳实践
4. 添加错误处理
```

### 5. 生成测试
```
为以下函数生成单元测试：
[粘贴函数代码]
要求：
1. 覆盖正常流程
2. 覆盖边界条件
3. 覆盖异常情况
4. 使用[Jest/pytest/etc]框架
```

## 四、提示词技巧

### 1. 提供上下文
```
项目背景：这是一个Express.js REST API
当前文件：src/routes/users.ts
依赖：Prisma ORM, Zod验证
```

### 2. 指定约束
```
不要使用any类型
不要使用console.log
错误处理使用自定义Error类
```

### 3. 给出示例
```
参考现有代码风格：
```typescript
export async function getUser(id: string): Promise<User> {
  const user = await prisma.user.findUnique({ where: { id } });
  if (!user) throw new NotFoundError('User not found');
  return user;
}
```
```

### 4. 分步要求
```
请分步实现：
1. 先定义类型接口
2. 实现核心逻辑
3. 添加错误处理
4. 添加日志记录
```

## 五、常见错误

### 错误1：需求太模糊
❌ "写一个登录功能"
✅ "用Express.js实现JWT登录接口，包含用户名密码验证、token生成、错误处理"

### 错误2：没有约束
❌ "用Python处理CSV"
✅ "用Python读取CSV文件，提取特定列，去重，按日期排序，输出为新CSV"

### 错误3：不审查代码
AI生成的代码必须审查：
- 安全性检查
- 逻辑正确性
- 边界条件
- 性能问题

## 六、总结

AI写代码的核心：**清晰的提示词 + 严格的审查 = 高质量代码**

从今天开始，用正确的姿势让AI帮你写代码。记住：AI是工具，你是工程师。

## 三、AI测试用例生成

### 基础流程
1. 提供被测代码
2. AI分析代码逻辑
3. 生成测试用例
4. 人工审查和调整

### 提示词示例
```
为以下函数生成单元测试：
[粘贴函数代码]

要求：
1. 覆盖正常流程
2. 覆盖边界条件
3. 覆盖异常情况
4. 使用[框架]风格
5. 包含测试描述
```

### 生成结果示例
```python
def test_calculate_discount_normal():
    """正常折扣计算"""
    assert calculate_discount(100, 10) == 90

def test_calculate_discount_zero():
    """零折扣"""
    assert calculate_discount(100, 0) == 100

def test_calculate_discount_full():
    """全折扣"""
    assert calculate_discount(100, 100) == 0

def test_calculate_discount_negative():
    """负折扣应抛出异常"""
    with pytest.raises(ValueError):
        calculate_discount(100, -10)
```

## 四、测试策略

### 测试金字塔
- **单元测试**（70%）：AI最容易生成
- **集成测试**（20%）：AI可以辅助
- **端到端测试**（10%）：AI辅助较少

### 建议
1. 用AI生成单元测试
2. 人工编写关键业务测试
3. AI辅助集成测试

## 五、常见问题

**Q：AI生成的测试够可靠吗？**
A：需要人工审查。AI擅长覆盖边界条件，但业务逻辑需要人工判断。

**Q：能提高测试覆盖率吗？**
A：可以。AI可以发现人工遗漏的边界条件。

**Q：需要修改代码吗？**
A：不需要。AI分析现有代码生成测试。

## 六、总结

AI测试用例生成的核心流程：提供代码 → AI分析 → 生成测试 → 人工审查
